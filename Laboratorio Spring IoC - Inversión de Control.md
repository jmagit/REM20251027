El valor que muestra despu√©s de la @ representa a la referencia, su valor concreto no es lo importante (lo puede cambiar el Garbage Collector), que coincidan o no si es relevante, dado que valores distintos representan instancias distintas. Cada vez que lo inyectes, Spring crear√° una nueva instancia.

### Con estado

Los beans *singleton* comparten su estado mientras que los *prototype* no.

`model/PrototypeBean.java`
```java
package com.example.model;

import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;

@Component
@Scope("prototype")
public class PrototypeBean {

    private static int instances = 0;
    private final int id;
    private int counter = 0;

    public PrototypeBean() {
        id = ++instances;
    }

    @PostConstruct
    public void init() {
        System.out.println("PrototypeBean init, id=" + id);
    }

    public int getNext() { 
    	return ++counter; 
    }
    
    @Override
    public String toString() {
        return "PrototypeBean#" + id + " counter: " + counter;
    }
}
```
